## Stage 1: Offer Design (`/solo:offer`)

### Purpose
Transform your expertise into packaged, sellable offers that scale beyond trading hours for dollars.

### Step 1: Expertise Audit

Map your skills across three dimensions. Score each 1-5.

```
+---------------------------+-----+------------+--------------+-------+
| Skill / Expertise         | Joy | Market     | Your Skill   | Total |
|                           | 1-5 | Demand 1-5 | Level 1-5    | /15   |
+---------------------------+-----+------------+--------------+-------+
| [e.g. Web development]    |     |            |              |       |
| [e.g. Copywriting]        |     |            |              |       |
| [e.g. Strategy consulting]|     |            |              |       |
| [e.g. Video production]   |     |            |              |       |
| [e.g. Data analysis]      |     |            |              |       |
+---------------------------+-----+------------+--------------+-------+
```

**Scoring Guide:**
- 12-15: Sweet spot — build your core offer here
- 9-11: Strong secondary offer or upsell
- 6-8: Only if market demand is 5/5
- Below 6: Drop it

### Step 2: Service-to-Product Ladder

Build a ladder from high-touch to scalable. Every solopreneur needs at least 3 rungs.

```
LADDER RUNG        | PRICE RANGE  | YOUR TIME  | SCALABILITY
-------------------+--------------+------------+------------
1:1 Done-for-you   | $$$$$        | High       | Low
1:1 Consulting      | $$$$         | High       | Low
Group coaching      | $$$          | Medium     | Medium
Online course       | $$           | Low (after)| High
Templates/tools     | $            | Minimal    | Very high
SaaS product        | $ recurring  | Dev upfront| Maximum
Free content        | $0           | Medium     | Infinite
```

**Action:** Place your current offers on this ladder. Identify the next rung to build.

### Step 3: Offer Packaging Template

For each offer, complete this template:

```
OFFER NAME: _______________________________________________
TARGET BUYER: _____________________________________________
PROBLEM SOLVED: ___________________________________________
DELIVERABLES:
  1. ____________________________________________________
  2. ____________________________________________________
  3. ____________________________________________________
TIMELINE: _________________________________________________
PRICE: ____________________________________________________
GUARANTEE: ________________________________________________
IDEAL NEXT STEP (upsell): _________________________________
```

### Step 4: Positioning Statement

Fill in the blanks:

> I help [TARGET AUDIENCE] achieve [SPECIFIC OUTCOME] through [YOUR METHOD/APPROACH], without [COMMON PAIN POINT THEY WANT TO AVOID].

**Anti-patterns:**
- "I help everyone" — too broad, you help no one
- "I do everything" — no positioning, no premium pricing
- Missing the "without" — you need to name the pain you remove
- No specific outcome — "grow their business" means nothing

### Offer Scoring Rubric

```
+-------------------------+---+---+---+---+---+
| Criterion               | 1 | 2 | 3 | 4 | 5 |
+-------------------------+---+---+---+---+---+
| Clear target buyer      |   |   |   |   |   |
| Specific outcome         |   |   |   |   |   |
| Defined deliverables    |   |   |   |   |   |
| Pricing justified       |   |   |   |   |   |
| Has a guarantee         |   |   |   |   |   |
| Leads to next offer     |   |   |   |   |   |
| Can explain in 1 sentence|   |   |   |   |   |
+-------------------------+---+---+---+---+---+
TOTAL: ___/35
```

- 30-35: Ship it
- 22-29: Needs tightening — sharpen the weakest area
- 15-21: Rethink fundamentals before launching
- Below 15: Start over with the expertise audit

## Stage 2: Revenue Diversification (`/solo:revenue`)

### Purpose
Build multiple income streams so no single client or product can sink your business.

### Step 1: Revenue Stream Audit

List every way you currently earn money.

```
+-------------------+--------+----------+-----------+--------+-----------+
| Revenue Stream    | Type   | Monthly  | Recurring | Active | % of Total|
|                   | A/P    | Revenue  | Y/N       | Hours  | Revenue   |
+-------------------+--------+----------+-----------+--------+-----------+
|                   |        |          |           |        |           |
+-------------------+--------+----------+-----------+--------+-----------+
```

Type: A = Active (requires your time), P = Passive (runs without you)

**Red flags:**
- Any single stream > 50% of revenue
- All streams are Active
- No recurring revenue
- Total hours > 40/week across all streams

### Step 2: Revenue Mix Targets by Stage

```
                    YEAR 1        YEAR 2        YEAR 3+
Active services     70%           50%           30%
Productized service 20%           25%           25%
Digital products    10%           15%           20%
Passive/recurring   0%            10%           25%
```

### Step 3: Stream Evaluation Scorecard

Score each potential new stream 1-5.

```
+-------------------+--------+--------+-------------+--------------+-------+
| Stream            | Effort | Margin | Scalability | Predictabil. | Total |
|                   | 1-5    | 1-5    | 1-5         | 1-5          | /20   |
+-------------------+--------+--------+-------------+--------------+-------+
|                   |        |        |             |              |       |
+-------------------+--------+--------+-------------+--------------+-------+
```

- 16-20: Build this next
- 12-15: Queue it for next quarter
- 8-11: Only if it complements existing streams
- Below 8: Skip

**Effort scoring (inverted — lower effort = higher score):**
- 5 = Set up once, earns forever (templates, affiliate)
- 4 = Monthly maintenance (membership, newsletter sponsorship)
- 3 = Batch work quarterly (course updates, content)
- 2 = Weekly time required (group coaching, retainers)
- 1 = Per-project, heavy time (1:1 services, custom work)

### Step 4: Diversification Roadmap

```
QUARTER | NEW STREAM TO ADD        | SETUP ACTIONS              | TARGET MONTHLY $
--------+--------------------------+----------------------------+------------------
Q1      |                          |                            |
Q2      |                          |                            |
Q3      |                          |                            |
Q4      |                          |                            |
```

**Anti-patterns:**
- Adding 3+ streams at once — you will finish none
- Chasing passive income before you have active income expertise
- Ignoring margins — $10K/mo at 20% margin is worse than $5K at 80%
- Diversifying before your core offer is proven

## Stage 3: Audience Building (`/solo:audience`)

### Purpose
Build an owned audience that you can monetize independent of any platform.

### Step 1: Platform Selection Matrix

Score each platform 1-5 for your situation.

```
+----------------+----------+----------+--------+--------------+-------+
| Platform       | Audience | Content  | Effort | Monetization | Total |
|                | Size 1-5 | Fit 1-5  | 1-5    | Potential 1-5| /20   |
+----------------+----------+----------+--------+--------------+-------+
| Newsletter     |          |          |        |              |       |
| LinkedIn       |          |          |        |              |       |
| Twitter/X      |          |          |        |              |       |
| YouTube        |          |          |        |              |       |
| Podcast        |          |          |        |              |       |
| Blog/SEO       |          |          |        |              |       |
| Instagram      |          |          |        |              |       |
| TikTok         |          |          |        |              |       |
+----------------+----------+----------+--------+--------------+-------+
```

**Rule: Pick 1 primary + 1 secondary. Newsletter is always primary (owned audience).**

### Step 2: Newsletter Growth Playbook

```
PHASE 1 (0-500 subscribers):
  - Lead magnet on your website (checklist, template, swipe file)
  - Share in every social bio
  - Cross-promote in communities you already belong to
  - Ask every client/contact to subscribe
  - Goal: 50 subscribers/month

PHASE 2 (500-2,000 subscribers):
  - Guest posts with CTA to newsletter
  - Newsletter swaps with similar-sized creators
  - Paid social ads to lead magnet ($5-10/day)
  - Referral program (3 referrals = bonus content)
  - Goal: 150 subscribers/month

PHASE 3 (2,000-10,000 subscribers):
  - Sponsorships (sell or buy)
  - Webinar partnerships
  - Product launches to the list
  - Segmentation and personalization
  - Goal: 500 subscribers/month
```

**Welcome sequence (5 emails):**
1. Deliver the lead magnet + set expectations
2. Your story + why you do this (day 2)
3. Best piece of content / quick win (day 4)
4. Social proof / case study (day 7)
5. Soft pitch to lowest-priced offer (day 10)

### Step 3: Content Flywheel

```
       CREATE
      (long-form)
         |
         v
    DISTRIBUTE -----> ENGAGE
    (repurpose)       (comments, DMs)
         ^                |
         |                v
         +--- CONVERT ----+
              (offer CTA)
```

Every piece of content should:
1. Demonstrate expertise (not just inform)
2. Include one actionable takeaway
3. End with a next step (follow, subscribe, buy)

### Step 4: 1000 True Fans Framework

```
YOUR TARGET:
  1,000 fans x $____ average annual spend = $_______ annual revenue

DEFINE A "TRUE FAN":
  - Buys everything you release
  - Shares your content unprompted
  - Would pay for early/premium access

HOW TO FIND THEM:
  - They comment consistently
  - They reply to emails
  - They refer others
  - They buy within 24 hours of a launch

HOW TO KEEP THEM:
  - Reply to every message
  - Give them exclusive access
  - Ask for their input on new offers
  - Surprise them (bonus content, handwritten notes)
```

**Anti-patterns:**
- Optimizing for followers instead of subscribers
- Being on 5 platforms, consistent on none
- Never asking for the sale
- Ignoring DMs and comments (engagement is the algorithm)

## Stage 4: Content System (`/solo:content`)

### Purpose
Create a repeatable content system that produces consistent output without daily creative decisions.

### Step 1: Content Batching Workflow

```
MONTHLY (1 day):
  - Choose 4 weekly themes aligned with your offers
  - Outline 4 long-form pieces (blog, video, podcast)
  - Identify key points for repurposing

WEEKLY (2 hours):
  - Write/record 1 long-form piece
  - Extract 5-7 short-form pieces from it
  - Schedule everything for the week

DAILY (15 minutes):
  - Post scheduled content
  - Reply to comments/DMs
  - Save ideas to swipe file
```

### Step 2: Repurposing Chain

```
LONG-FORM CONTENT (1 piece)
  |
  +---> Newsletter edition
  +---> 3-5 social posts (key points)
  +---> 1 thread (step-by-step breakdown)
  +---> 2-3 stories (behind the scenes, polls)
  +---> 1 email to list (different angle)
  +---> Quote graphics (pull best lines)
  +---> Comment responses (expand on points)
```

**1 piece of content = 10-15 pieces of distribution.**

### Step 3: Content Calendar Template (4-Week Rolling)

```
+------+----------+-----------+-----------+-----------+-----------+
| Week | Theme    | Long-Form | Social    | Email     | CTA       |
+------+----------+-----------+-----------+-----------+-----------+

+------+----------+-----------+-----------+-----------+-----------+

+------+----------+-----------+-----------+-----------+-----------+

### Step 4: Content Types by Funnel Stage

```
AWARENESS (top):
  - How-to guides
  - Industry hot takes
  - Data/research breakdowns
  - "Myth vs reality" posts

CONSIDERATION (middle):
  - Case studies

- Creating only top-of-funnel content (lots of likes, no sales)

- Random topics with no connection to your offers

## Stage 5: Sales Copy (`/solo:copy`)

### Purpose
Write copy that converts visitors into buyers using proven frameworks.

### Step 1: Landing Page Formula (PAS)

```
SECTION 1 — PROBLEM
  Headline: Name the painful problem they recognize
  Subheadline: Agitate with specifics ("You've tried X but still...")
  Bullet points: 3-5 symptoms they're experiencing

SECTION 2 — AGITATE
  Story/scenario: Paint the picture of what happens if they don't solve this
  Stakes: What they're losing (money, time, opportunity)
  Empathy: Show you understand ("I've been there")

SECTION 3 — SOLVE
  Your offer: What it is and what it does for them
  Benefits (not features): 3-5 outcomes they'll get
  Social proof: Testimonials, logos, numbers
  CTA: Clear, specific, low-friction

SECTION 4 — PROOF + CLOSE
  FAQ: Handle top 3-5 objections
  Guarantee: Remove risk
  Final CTA: Repeat with urgency
```

### Step 2: Email Copy Frameworks

**AIDA (Awareness, Interest, Desire, Action):**
```
Subject line:  Grab attention (curiosity, benefit, or urgency)
Line 1:        Hook — open loop or bold claim
Body:          Build interest with story or data
Desire:        Paint the outcome they want
CTA:           One clear action (reply, click, buy)
```

**Before-After-Bridge:**
```
BEFORE:  "Right now you're dealing with [pain]..."
AFTER:   "Imagine if [desired outcome]..."
BRIDGE:  "Here's how to get there: [your offer]"
```

### Step 3: Ad Copy Templates

**Headline Formulas:**
```
1. How to [achieve outcome] without [pain point]
2. [Number] ways to [achieve outcome] in [timeframe]
3. Stop [bad habit]. Start [good habit]. Here's how.
4. The [adjective] guide to [topic] for [audience]
5. Why [common approach] is costing you [loss]
```

**CTA Patterns:**
```
LOW COMMITMENT:  "Get the free guide" / "See how it works"
MEDIUM:          "Book a free call" / "Start your trial"
HIGH:            "Buy now" / "Enroll today" / "Hire me"
```

### Step 4: Copy Scoring Rubric

```
+-------------------------+---+---+---+---+---+
| Criterion               | 1 | 2 | 3 | 4 | 5 |
+-------------------------+---+---+---+---+---+
| Clarity (no jargon)     |   |   |   |   |   |
| Urgency (why now)       |   |   |   |   |   |
| Proof (social evidence) |   |   |   |   |   |
| Benefit-led (not feature)|  |   |   |   |   |
| Single clear CTA        |   |   |   |   |   |
| Reads at 8th grade level|   |   |   |   |   |
| Addresses objections    |   |   |   |   |   |
+-------------------------+---+---+---+---+---+
TOTAL: ___/35
```

- 30-35: Publish it
- 22-29: Tighten the weak spots
- 15-21: Rewrite with a specific framework
- Below 15: You are describing features, not selling outcomes

**Anti-patterns:**
- Writing about yourself instead of the reader's problem
- Multiple CTAs competing for attention
- No social proof anywhere on the page
- "Click here" as your CTA text (say what they get)

## Stage 7: Sales (`/solo:sales`)

### Purpose
Close deals consistently without feeling salesy. Structured process beats improvisation.

### Step 1: Discovery Call Script

```
OPEN (2 min):
  "Thanks for taking the time. To make sure this is a good fit,
   I'll ask a few questions, share how I might help, and we'll
   decide together if it makes sense to move forward. Sound good?"

QUALIFY (5 min):
  1. "What's the main challenge you're trying to solve?"
  2. "How long has this been a problem?"
  3. "What have you tried so far?"
  4. "What happens if you don't solve this in the next 3-6 months?"

DIAGNOSE (5 min):
  5. "What does success look like for you?"
  6. "What's your timeline?"
  7. "Is there a budget range you're working within?"
  8. "Who else is involved in this decision?"

PRESCRIBE (5 min):
  "Based on what you've shared, here's what I'd recommend..."
  [Explain your approach — not a full proposal, just the framework]
  "I'll put together a detailed proposal by [date]. Does that work?"

CLOSE (3 min):
  "Any questions before we wrap up?"
  "I'll send the proposal by [date]. If it looks good, we can
   kick off as early as [date]."
```

### Step 2: Proposal Template

```
1. EXECUTIVE SUMMARY
   - Problem: [their words, not yours]
   - Goal: [specific outcome]
   - Approach: [your method in 2-3 sentences]

2. SCOPE OF WORK
   - Deliverable 1: [description] — [timeline]
   - Deliverable 2: [description] — [timeline]
   - Deliverable 3: [description] — [timeline]
   - Out of scope: [explicitly list what's NOT included]

3. INVESTMENT
   - Option A: [base package] — $X
   - Option B: [recommended] — $Y (includes Z)
   - Option C: [premium] — $Z (includes everything + bonus)
   - Payment terms: [50% upfront, 50% on completion / monthly]

4. TIMELINE
   - Start: [date]
   - Milestones: [key dates]
   - Completion: [date]

5. NEXT STEPS
   - Sign this proposal
   - Submit deposit
   - Complete onboarding questionnaire
```

### Step 3: Follow-Up Sequence (5-Touch Framework)

```
TOUCH | TIMING        | CHANNEL | MESSAGE
------+---------------+---------+------------------------------------------
1     | Same day      | Email   | "Great call — here's the proposal"
2     | Day 3         | Email   | "Any questions about the proposal?"
3     | Day 7         | Phone   | Quick voicemail or text
4     | Day 14        | Email   | Share relevant case study or resource
5     | Day 21        | Email   | "Closing the loop" — final check-in
```

**After touch 5 with no response:** Move to nurture list. Do not chase.

### Step 4: Objection Handling

```
OBJECTION        | RESPONSE FRAMEWORK
-----------------+------------------------------------------------------
"Too expensive"  | "I understand. Can I ask what budget you had in mind?
                 |  [Then: adjust scope, offer payment plan, or show ROI]"
                 |
"Bad timing"     | "When would be better? I'll follow up then.
                 |  [Book a specific date — don't say 'whenever']"
                 |
"Need to think"  | "Of course. What specifically do you want to think
                 |  through? [Identify the real objection underneath]"
                 |
"Can do it myself"| "You absolutely could. Most of my clients could too.
                 |  They hire me for speed and to avoid costly mistakes."
                 |
"Need approval"  | "Happy to help. Want me to join a call with [person]
                 |  or prepare a summary they can review?"
```

### Step 5: Close Rate Benchmarks

```
OFFER TYPE          | HEALTHY CLOSE RATE | RED FLAG
--------------------+--------------------+---------
Free discovery call | 20-30% → paid      | < 10%
Low-ticket ($50-500)| 5-15% of visitors  | < 2%
Mid-ticket ($1-5K)  | 15-25% of calls    | < 10%
High-ticket ($5K+)  | 25-40% of calls    | < 15%
```

**Anti-patterns:**
- Sending proposals without a discovery call
- Not following up (80% of sales happen after touch 5)
- Discounting instead of adjusting scope
- Talking more than listening on calls

## Stage 8: Client Onboarding (`/solo:onboard`)

### Purpose
Set the tone for a successful engagement from day one. Good onboarding prevents 90% of client problems.

### Step 1: Onboarding Checklist Template

```
[ ] Contract/agreement signed
[ ] Payment received (deposit or first invoice)
[ ] Onboarding questionnaire sent and completed
[ ] Welcome packet delivered
[ ] Kickoff call scheduled
[ ] Project management access shared (if applicable)
[ ] Communication channels established
[ ] First milestone defined and confirmed
[ ] Internal project setup (files, folders, tracking)
```

### Step 2: Welcome Packet Contents

```
1. Welcome message (personal, enthusiastic)
2. What to expect (process overview, timeline)
3. Communication guidelines:
   - Preferred channel: [email / Slack / portal]
   - Response time: [24 hours on weekdays]
   - Meeting cadence: [weekly / biweekly]
4. Your role vs. their role (who does what)
5. Key dates and milestones
6. FAQ (top 5 questions new clients ask)
7. How to share feedback
8. Emergency contact process
```

### Step 3: Expectation Setting

```
COVER THESE IN THE KICKOFF CALL:

SCOPE:
  "Here's exactly what's included: [list deliverables]
   And here's what's outside scope: [list exclusions]
   If you need something not listed, we'll discuss it as a
   change request with a separate quote."

COMMUNICATION:
  "I check messages [frequency]. Expect responses within
   [timeframe]. For urgent items, [method]."

TIMELINE:
  "Here's our milestone schedule: [dates]
   I'll need [specific inputs] from you by [dates] to stay
   on track. Delays on inputs shift the timeline equally."

REVISIONS:
  "This engagement includes [N] rounds of revisions.
   Additional rounds are billed at [rate]."
```

### Step 4: Automation Opportunities

```
TASK                    | MANUAL TIME  | TOOL TO AUTOMATE
------------------------+--------------+------------------
Scheduling calls        | 15 min each  | Calendly / Cal.com
Sending contracts       | 20 min       | HelloSign / PandaDoc
Collecting info         | 30 min       | Typeform / Google Form
Sending welcome packet  | 15 min       | Email template + CRM
Payment collection      | 10 min       | Stripe / invoice auto
Project setup           | 30 min       | Template in PM tool
Status updates          | 20 min/week  | Automated from PM tool
```

**Total time saved per client: 2-3 hours.**

**Anti-patterns:**
- No written scope (guaranteed scope creep)
- Skipping the kickoff call (misaligned expectations)
- Manual processes for every new client (doesn't scale)

## Stage 10: Customer Support (`/solo:support`)

### Purpose
Handle support efficiently so it doesn't consume your entire week. Self-service first, personal touch when it matters.

### Step 1: FAQ Template Design

```
FAQ STRUCTURE (aim for 10-20 questions):

BEFORE BUYING:
  Q: What's included in [offer]?
  Q: How long does [process] take?
  Q: What results can I expect?
  Q: Do you offer refunds?

DURING ENGAGEMENT:
  Q: How do I contact you?
  Q: What if I need changes?
  Q: What if I'm not happy with the work?

AFTER DELIVERY:
  Q: Do you offer ongoing support?
  Q: Can I hire you again for [next thing]?
  Q: How do I leave a review/testimonial?
```

### Step 2: Response Templates (Common Scenarios)

```
NEW INQUIRY:
  "Thanks for reaching out! I'd love to learn more about
   your project. Could you share:
   1. What you're looking to accomplish
   2. Your timeline
   3. Your budget range
   I'll get back to you within 24 hours."

SCOPE QUESTION:
  "Great question. [Answer]. This is covered in the
   [package/tier]. If you need [additional thing],
   we can discuss that as an add-on."

DELAY NOTIFICATION:
  "I wanted to give you a heads up — [deliverable] will
   be [N days] behind schedule due to [reason]. The new
   delivery date is [date]. I'm [action to minimize impact]."

TESTIMONIAL REQUEST:
  "It's been great working with you! Would you be open to
   sharing a brief testimonial? A few sentences about
   [specific result] would be incredibly helpful.
   Here's a quick form: [link]"

REFUND REQUEST:
  "I'm sorry to hear that. I want to make this right.
   [If within policy: Here's how the refund process works.]
   [If outside policy: Let me see what I can do. Could you
   share more about what didn't meet your expectations?]"
```

### Step 3: Escalation Matrix

```
LEVEL 1 — SELF-SERVICE (no time cost):
  FAQ page, knowledge base, automated emails
  Example: "How do I access my course?"

LEVEL 2 — ASYNC (5 min per ticket):
  Email response using templates
  Example: "My download link isn't working"

LEVEL 3 — SYNC (15-30 min):
  Video call or screen share
  Example: "I don't understand how to implement step 3"

LEVEL 4 — RESOLUTION (varies):
  Refund, redo, or escalation to external help
  Example: "This isn't what I expected at all"
```

**Route as many requests as possible to Level 1.**

### Step 4: Support Metrics

```
METRIC              | TARGET        | MEASURE HOW
--------------------+---------------+------------------------------
Response time       | < 24 hours    | Time from request to first reply
Resolution time     | < 48 hours    | Time from request to resolved
First-contact fix   | > 70%         | Resolved without follow-up
CSAT                | > 4.5/5       | Post-resolution survey
Support volume      | Decreasing    | Tickets per month (track trend)
Self-service rate   | > 50%         | FAQ views / total requests
```

**Anti-patterns:**
- No FAQ (answering the same question 50 times)
- No templates (writing every response from scratch)
- Responding to support instead of building self-service
- No escalation path (everything gets a 30-min call)
