# solo-skills

Run a one-person company like a team of ten.

Strategy, sales, operations, and finances — all without hiring. A pure markdown skill for [Claude Code](https://claude.com/claude-code).

## Install

```bash
npx skills add ORBWEVA/solo-skills
```

## Commands

| Command | Description |
|---------|-------------|
| `/solo:offer` | Productize expertise into sellable offers (service to product ladder) |
| `/solo:revenue` | Revenue diversification (consulting + products + passive income) |
| `/solo:audience` | Audience building system (newsletter, social, content flywheel) |
| `/solo:content` | Content system (batching, repurposing, distribution calendar) |
| `/solo:copy` | Sales copy frameworks (landing pages, emails, ads) |
| `/solo:leads` | Lead generation (funnels, magnets, referrals, partnerships) |
| `/solo:sales` | Solo sales playbook (proposals, follow-ups, closing) |
| `/solo:onboard` | Client onboarding automation |
| `/solo:clients` | Client management (communication, scope, deliverables) |
| `/solo:support` | Customer support system (FAQ, templates, escalation) |
| `/solo:books` | Bookkeeping + P&L + expense tracking |
| `/solo:tax` | Tax prep checklist (quarterly estimates, deductions) |
| `/solo:time` | Time audit + priority matrix (keep/cut/automate) |
| `/solo:stack` | Tech stack builder (tools, costs, integrations) |
| `/solo:help` | Command reference |

## The Flow

Each command is a standalone stage, but they connect into a natural workflow:

```
OFFER ──> REVENUE ──> AUDIENCE ──> CONTENT ──> LEADS
                                                 |
                                                 v
STACK <── TIME <── BOOKS <── CLIENTS <── SALES <─┘
                     |                    |
                     v                    v
                    TAX               ONBOARD ──> SUPPORT
```

**Start here:**
- New solopreneur? Start with `/solo:offer` to package your expertise
- Already selling? Run `/solo:time` to find where hours are leaking
- Want more clients? Use `/solo:leads` then `/solo:sales`
- Need systems? Try `/solo:stack` then `/solo:content`

## Principles

1. **Revenue before vanity metrics** — followers mean nothing if nobody buys
2. **Systems over hustle** — build once, run forever
3. **Automate the repeatable, humanize the valuable** — your time goes to high-judgment work
4. **One person, ten systems** — you don't need a team, you need processes
5. **Profit margin > revenue** — $5K/mo at 80% margin beats $15K at 20%
